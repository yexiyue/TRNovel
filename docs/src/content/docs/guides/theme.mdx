TRNovel 的界面外观现在由 `ratatui-kit-themes` 的命名主题驱动。主页选择「主题设置」进入后，可以在完整主题之间切换，而不是逐项编辑六个颜色。

![主题设置](../../../assets/guides/theme.gif)

## 新主题系统做了什么

旧版主题页是六个颜色字段的手动编辑器。新版改为完整命名主题：每个主题都带有背景、正文、边框、选中态、成功 / 警告 / 错误 / 信息等语义色，TRNovel 的页面、列表、弹窗、阅读器和 TTS 设置会一起跟随当前主题变化。

主题列表中的每一项会显示主题名称、保存用的 slug，以及一组关键色板预览。这样切换主题时能直接看到主要色彩，而不需要逐项记忆颜色值。

## 选择命名主题

主题设置页会列出所有内置主题，每项显示主题名称、保存用的 slug，以及一组关键色板预览。用 `↑/↓` 或 `j/k` 移动，按 `Enter` 应用当前主题。

选择会立即生效，并保存到 `~/.novel/appearance.json`。配置只记录主题 slug 与背景模式，不保存派生后的完整色板。

## 背景模式

按 `t` 可以在两种背景模式之间切换：

| 模式 | 行为 |
|------|------|
| 终端背景 | 保留终端本身的背景色，只应用主题的文字、边框、高亮和语义色 |
| 主题背景 | 使用主题自带背景色填充 TRNovel 根界面 |

背景模式同样会立即保存并应用。

## 快捷键

| 快捷键 | 行为 |
|--------|------|
| `↑/↓` 或 `j/k` | 移动主题列表 |
| `Enter` | 应用当前选中的主题 |
| `t` | 切换终端背景 / 主题背景 |
| `d` | 打开恢复默认确认框 |
| `i` | 查看当前页面快捷键 |

## 恢复默认

在主题设置页按 `d` 弹出确认框。确认后会恢复默认外观：`tokyo-night` 主题与「终端背景」模式。

## 旧配置兼容性

旧版本的六色编辑流程已经移除。`~/.novel/theme.json` 不再被读取、迁移或保存；如果该文件仍然存在，新主题系统会忽略它。用户在新设置页保存外观后，会写入新的 `~/.novel/appearance.json`。

如果你以前依赖手写 HEX 调整单个颜色，需要改为选择最接近的命名主题。这样所有组件都能从同一套语义色派生样式，避免出现某些页面仍沿用旧颜色、阅读器和弹窗不一致的问题。
